## Partie 1 — Recrutement

### 1.1 Le Tableau Kanban des Candidats

**Accès** : Menu → Recrutement → **Candidats**

Le Kanban est un tableau à **4 colonnes** qui représente les étapes du recrutement :

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│    REÇUS     │  │  ENTRETIEN   │  │   RECRUTÉS   │  │   REFUSÉS    │
│              │  │              │  │              │  │              │
│  Candidat A  │  │  Candidat C  │  │  Candidat E  │  │  Candidat F  │
│  Candidat B  │  │  Candidat D  │  │              │  │              │
│              │  │              │  │              │  │              │
└──────────────┘  └──────────────┘  └──────────────┘  └──────────────┘
```

**Actions** :
- **Nouveau candidat** : Bouton **+ Nouveau candidat** en haut à droite
- **Changer de colonne** : Glisser-déposer une fiche d'une colonne à l'autre
- **Voir le détail** : Cliquer sur le nom du candidat

### 1.2 Fiche Candidat

En cliquant sur un candidat, vous accédez à sa fiche complète :

| Section | Contenu |
|---------|---------|
| **Informations** | Nom, prénom, téléphone, email, adresse |
| **CV** | Télécharger un CV (PDF/Word) — le système extrait automatiquement les compétences |
| **Compétences** | Compétences détectées depuis le CV + manuelles |
| **Entretiens** | Historique des entretiens avec grille d'évaluation |
| **Mise en situation** | Résultats des exercices pratiques |
| **Documents** | CV, lettres, attestations, pièces d'identité |
| **Historique** | Tous les changements de statut avec dates |
| **Notes** | Commentaires libres |

### 1.3 Planifier un Entretien

1. Ouvrir la fiche du candidat
2. Cliquer sur l'onglet **Entretiens**
3. Cliquer sur **Planifier un entretien**
4. Remplir :
   - Date et heure
   - Type (téléphonique, présentiel, mise en situation)
   - Intervieweur
5. Après l'entretien, **saisir l'évaluation** :
   - Grille structurée (motivation, compétences, savoir-être)
   - Note globale
   - Recommandation (recruter / entretien supplémentaire / refuser)

### 1.4 Plan de Recrutement

Sur la page Candidats, un onglet **Plan de recrutement** permet de :
- Définir les objectifs par filière (tri, collecte, logistique, boutique)
- Suivre les postes ouverts vs pourvus
- Planifier les besoins futurs

### 1.5 Recruter un Candidat

Quand un candidat passe dans la colonne **Recrutés** :
- Le système propose de **créer sa fiche collaborateur** automatiquement
- Les informations personnelles sont pré-remplies
- Vous complétez : contrat, équipe, poste, date de début

## Partie 2 — Test PCM

### 2.1 Qu'est-ce que le PCM ?

Le **Process Communication Model** est un test de personnalité en **20 questions** qui identifie 6 types :

| Type | Caractéristiques | Communication adaptée |
|------|-----------------|---------------------|
| **Empathique** | Chaleureux, sensible, attentif | Reconnaissance personnelle, ambiance |
| **Travaillomane** | Logique, organisé, factuel | Données, faits, planification |
| **Persévérant** | Engagé, observateur, consciencieux | Opinions, valeurs, convictions |
| **Rêveur** | Calme, imaginatif, réflexif | Directives claires, temps de réflexion |
| **Promoteur** | Action, défi, adaptation | Défis, résultats, autonomie |
| **Rebelle** | Créatif, spontané, ludique | Humour, jeu, réactions |

### 2.2 Faire Passer un Test

**Accès** : Menu → Recrutement → **Matrice PCM**

1. Cliquer sur **Nouveau test**
2. Sélectionner le collaborateur
3. Le collaborateur répond aux **20 questions** (choix multiple)
4. Le système calcule automatiquement le profil

### 2.3 Lire les Résultats

Le résultat se présente sous forme d'un **immeuble** (métaphore visuelle) :

```
┌─────────────┐
│   Étage 6   │  ← Type le moins développé
├─────────────┤
│   Étage 5   │
├─────────────┤
│   Étage 4   │
├─────────────┤
│   Étage 3   │
├─────────────┤
│   PHASE     │  ← Mode de fonctionnement ACTUEL
├─────────────┤
│   BASE      │  ← Type DOMINANT (fondation de la personnalité)
└─────────────┘
```

- **Base** (fondation) : Le type dominant, stable toute la vie
- **Phase** (étage actuel) : Le mode de fonctionnement actuel (peut changer)
- **Profil de stress** : Signaux d'alerte quand la personne est sous pression

### 2.4 Utiliser le PCM au Quotidien

| Situation | Comment utiliser le PCM |
|-----------|----------------------|
| **Entretien d'insertion** | Adapter votre langage au type dominant |
| **Conflit** | Identifier le canal de communication approprié |
| **Stress** | Reconnaître les signaux et adapter votre réponse |
| **Motivation** | Activer le bon levier selon le type |
| **Équipe** | Composer des binômes complémentaires |

**Export PDF** : Cliquer sur **Exporter PDF** pour un document A4 imprimable.

## Partie 3 — Gestion des Collaborateurs

### 3.1 Liste des Collaborateurs

**Accès** : Menu → Gestion Équipe → **Collaborateurs**

- Tableau avec tous les collaborateurs actifs
- **Filtres** : par équipe, type de contrat, statut
- **Recherche** : par nom ou prénom

### 3.2 Fiche Collaborateur

| Section | Contenu |
|---------|---------|
| **Identité** | Nom, prénom, date naissance, coordonnées |
| **Contrat** | Type (CDI/CDD/CDDI), dates, renouvellements |
| **Poste** | Équipe, filière, poste actuel |
| **Compétences** | Liste avec niveaux (débutant → expert) |
| **Parcours insertion** | Lien vers le suivi CDDI (si applicable) |
| **Historique** | Événements, changements de poste, absences |

### 3.3 Types de Contrats

| Type | Durée | Spécificité |
|------|-------|-------------|
| **CDI** | Indéterminée | Contrat permanent |
| **CDD** | Déterminée | Contrat temporaire classique |
| **CDDI** | Max 24 mois | Contrat d'insertion (renouvellement possible) |

**Pour les CDDI** : le système alerte automatiquement **30 jours avant** l'échéance du contrat.

## Partie 4 — Suivi des Heures

### 4.1 Saisie des Heures

**Accès** : Menu → Gestion Équipe → **Heures de travail**

1. Sélectionner la **semaine** ou le **mois** concerné
2. Pour chaque collaborateur et chaque jour :
   - Saisir les **heures normales**
   - Saisir les **heures supplémentaires** (si applicable)
   - Indiquer les **absences** (maladie, congé, formation...)
3. **Valider** la saisie

### 4.2 Validation

- Vérifier la cohérence des saisies
- Le système alerte en cas d'incohérence (ex : plus de 10h/jour)
- Export possible pour la paie

## Partie 5 — Compétences

### 5.1 Matrice de Compétences

**Accès** : Menu → Gestion Équipe → **Compétences**

La matrice croise **collaborateurs × compétences** :

```
                  Tri qualité  Recyclage  Conduite  Manutention  Informatique
Collaborateur A   ████████     ████       ──        ████████     ████
Collaborateur B   ████         ████████   ████████  ████████     ──
```

**Niveaux** :
- Débutant (1) → Intermédiaire (2) → Confirmé (3) → Expert (4)

### 5.2 Ajouter / Modifier une Compétence

1. Cliquer sur la cellule croisement collaborateur × compétence
2. Sélectionner le niveau
3. Ajouter une date d'évaluation

- Les **besoins de formation** (niveaux bas dans des compétences critiques)

## Partie 6 — Parcours d'Insertion (Module Clé)

### 6.1 Qu'est-ce que le Parcours d'Insertion ?

Le parcours d'insertion concerne les **salariés en CDDI** (Contrat d'Insertion).

Objectif : **lever les freins à l'emploi durable** en accompagnant chaque personne individuellement pendant la durée de son contrat (max 24 mois).

```
Entrée CDDI → Diagnostic → Jalon M1 → Jalon M6 → Jalon M12 → Sortie
                (7 freins)    (1 mois)   (6 mois)   (12 mois)
```

**3 jalons obligatoires** : M1 (1 mois), M6 (6 mois), M12 (12 mois)

### 6.2 Le Diagnostic Initial

**Accès** : Menu → Gestion Équipe → **Parcours insertion** → Sélectionner le collaborateur

Le diagnostic évalue **7 freins périphériques** :

| # | Frein | Exemples |
|---|-------|----------|
| 1 | **Logement** | Sans domicile, hébergement précaire, logement insalubre |
| 2 | **Mobilité** | Pas de permis, pas de véhicule, transport en commun limité |
| 3 | **Santé** | Problèmes de santé, addictions, suivi médical insuffisant |
| 4 | **Administratif** | Papiers manquants, droits non ouverts, dettes admin |
| 5 | **Financier** | Surendettement, absence de compte bancaire, dettes |
| 6 | **Famille** | Garde d'enfants, violences, isolement familial |
| 7 | **Justice** | Casier judiciaire, contrôle judiciaire, obligations légales |

**Notation** : de **1** (bloquant) à **5** (résolu) pour chaque frein

Le résultat est affiché sous forme de **radar** (diagramme en étoile) :

```
              Logement (4)
                 ╱╲
    Justice    ╱    ╲   Mobilité
      (3)    ╱   ████ ╲   (2)
           ╱  ████████  ╲
         ╱  ████████████  ╲
Famille ────████████████████──── Santé
  (4)     ╲  ████████████  ╱     (3)
           ╲  ████████  ╱
    Financier ╲  ████ ╱  Administratif
      (3)       ╲  ╱       (4)
                 ╲╱
```

### 6.3 Les Jalons M1 / M6 / M12

À chaque jalon, le CIP (Conseiller en Insertion Professionnelle) :

1. **Programme un entretien** avec le collaborateur
2. **Met à jour le radar** des 7 freins (nouvelle notation)
3. **Fait le bilan** des actions réalisées depuis le dernier jalon
4. **Compare** avec le diagnostic initial (évolution positive/négative)
5. **Définit** les objectifs pour la période suivante

**Le système affiche l'évolution** :

```
Frein Mobilité :  Diagnostic: 2 → M1: 2 → M6: 3 → M12: 4  ✅ Progression
Frein Santé :     Diagnostic: 3 → M1: 3 → M6: 2 → M12: 3  ⚠️ Stagnation
```

### 6.4 Plans d'Action CIP

Pour chaque frein identifié, le CIP crée des **actions concrètes** :

| Champ | Description | Exemple |
|-------|------------|---------|
| **Frein** | Le frein concerné | Mobilité |
| **Action** | L'action à réaliser | Inscription auto-école |
| **Responsable** | Qui s'en charge | CIP + collaborateur |
| **Échéance** | Date limite | 15 avril 2026 |
| **Statut** | État d'avancement | À faire / En cours / Fait |

**Exemples de plans d'action par frein** :

| Frein | Actions types |
|-------|-------------|
| **Mobilité** | Auto-école, aide au permis, covoiturage, abonnement transport |
| **Logement** | Demande logement social, aide au 1er loyer, hébergement d'urgence |
| **Santé** | Médecin traitant, bilan de santé, suivi addictologie, CPAM |
| **Administratif** | Renouvellement papiers, ouverture droits, accompagnement CAF |
| **Financier** | Dossier surendettement, ouverture compte, micro-crédit |
| **Famille** | Aide à la garde d'enfants, médiation familiale |
| **Justice** | Aide juridictionnelle, suivi SPIP, réhabilitation |

### 6.5 Alertes Entretiens

Le système génère des **alertes automatiques** :

| Alerte | Quand | Action |
|--------|-------|--------|
| **Jalon M1 approche** | 7 jours avant la date | Programmer l'entretien |
| **Jalon M6 approche** | 14 jours avant | Programmer l'entretien |
| **Jalon M12 approche** | 14 jours avant | Programmer l'entretien |
| **Entretien non programmé** | Quand un jalon est dépassé | Programmer en urgence |
| **Action CIP en retard** | Quand une action dépasse l'échéance | Relancer |
| **Contrat CDDI expire** | 30 jours avant | Décider (renouvellement ou sortie) |

### 6.6 Indicateurs d'Insertion

| Indicateur | Ce qu'il mesure | Objectif |
|-----------|----------------|---------|
| **Nombre de parcours actifs** | CDDI en cours | — |
| **Taux de jalons atteints** | Jalons M1/M6/M12 réalisés à temps | > 90% |
| **Évolution radar** | Progression moyenne des 7 freins | Positive |
| **Actions réalisées** | % des plans CIP terminés | > 70% |
| **Taux de sortie dynamique** | Emploi, formation, création après CDDI | > 50% |

## Partie 7 — Planning Hebdomadaire

### 7.1 Créer le Planning

**Accès** : Menu → Gestion Équipe → **Planning hebdo**

Le planning répartit les collaborateurs dans **4 filières** :

| Filière | Description |
|---------|-------------|
| **Tri** | Travail sur les chaînes de tri |
| **Collecte** | Tournées de collecte (chauffeurs) |
| **Logistique** | Préparation, chargement, expéditions |
| **Boutiques** | Vente en boutique solidaire |

**Pour chaque jour de la semaine** :
1. Sélectionner la semaine avec le sélecteur de date
2. Pour chaque collaborateur, choisir sa filière du jour
3. Vérifier les **disponibilités** (congés, absences, formations)
4. Valider le planning

## Partie 9 — RGPD

### 9.1 Vos Obligations

En tant que gestionnaire RH, vous manipulez des **données personnelles sensibles**. Le module RGPD de SOLIDATA vous aide à respecter le Règlement :

**Accès** : Menu → Administration → **RGPD**

| Fonctionnalité | Description |
|---------------|-------------|
| **Registre des traitements** | Liste de tous les traitements de données personnelles |
| **Consentements** | Suivi des consentements des collaborateurs |
| **Audit log** | Historique de tous les accès et modifications aux données |
| **Anonymisation** | Anonymiser les données d'un collaborateur après son départ |

### 9.2 Bonnes Pratiques RGPD

- Ne consulter que les données **nécessaires** à votre mission
- Ne pas partager les résultats PCM sans accord du collaborateur
- **Anonymiser** les données des collaborateurs partis (après le délai légal)
- Les données d'insertion sont **confidentielles** — ne les partager qu'avec les CIP habilités

## Partie 10 — Bonnes Pratiques

### Routine Quotidienne

| Action | Écran |
|--------|-------|
| □ Vérifier les alertes (jalons, contrats, entretiens) | Dashboard |
| □ Traiter les nouvelles candidatures | Candidats (Kanban) |
| □ Valider les heures de la veille | Heures de travail |

### Routine Hebdomadaire

| Action | Écran |
|--------|-------|
| □ Mettre à jour le planning de la semaine suivante | Planning hebdo |
| □ Programmer les entretiens d'insertion à venir | Parcours insertion |
| □ Suivre les plans d'action CIP en cours | Parcours insertion |
| □ Faire le point sur les recrutements en cours | Candidats |

### Routine Mensuelle

| Action | Écran |
|--------|-------|
| □ Analyser le reporting RH | Reporting RH |
| □ Bilan des parcours insertion (évolution radars) | Parcours insertion |
| □ Mettre à jour la matrice de compétences | Compétences |
| □ Vérifier les contrats arrivant à échéance | Collaborateurs |

### Routine Trimestrielle

| Action | Écran |
|--------|-------|
| □ Préparer les données Refashion (DPAV) | Refashion |
| □ Bilan d'insertion trimestriel | Reporting RH + Insertion |
| □ Revue du plan de recrutement | Candidats |
| □ Audit RGPD (vérification des consentements) | RGPD |

### Conseils Clés

1. **Les jalons d'insertion sont obligatoires** — Ne laissez pas passer les échéances M1/M6/M12
2. **Le PCM est un outil puissant** — Utilisez-le pour adapter votre communication avec chaque collaborateur
3. **Les plans d'action CIP doivent être concrets** — Chaque action a un responsable et une échéance
4. **Le radar des 7 freins** est votre boussole — Il montre la progression de chaque parcours
5. **Documentez tout** — L'historique dans SOLIDATA est votre preuve de suivi (audits, financeurs)
6. **Respectez la confidentialité** — Les données d'insertion sont sensibles (RGPD)
